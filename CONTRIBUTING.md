# Contributing to MeltPoolDG

MeltPoolDG is an open-source software project developed in an open and accessible environment.
Contributions from the community are highly encouraged, and anyone interested in using or extending the software is welcome to participate.

This document outlines the guidelines for contributing to MeltPoolDG.

## How can I contribute?

Contributions to MeltPoolDG should be submitted through *pull requests* on the project's
[official GitHub repository](https://github.com/MeltPoolDG/MeltPoolDG-dev).
We recommend creating a personal *fork* of the repository and developing changes on a dedicated *branch*.
Once your work is ready, you can propose its inclusion in the main code base by opening a pull request,
allowing other developers to review and discuss the proposed changes.

Before opening a pull request, please rebase your branch onto the latest master branch.
This helps maintain a clean, linear project history and simplifies the merging process.
Contributors are responsible for ensuring that their pull requests are well-prepared and
straightforward to review.

Comprehensive instructions on how to contribute to the MeltPoolDG code base are provided in our
[contribution guide](https://meltpooldg.github.io/MeltPoolDG/contributing.html#making-changes-and-pushing-to-the-repository).

To facilitate the review process, we provide a pull request description template that includes a checklist
of items to verify before submission. Contributors are also encouraged to use meaningful commit messages
and follow the guidelines described [here](https://www.conventionalcommits.org/en/v1.0.0/).
As a general rule, pull requests should be kept reasonably small, ideally no more than approximately 1000
added or modified lines of code, as larger changes can significantly increase the review effort.

To maintain consistency throughout the code base, all contributions should adhere to the MeltPoolDG
coding standards. In particular, source code formatting should be handled using `clang-format`,
and contributors should follow our
[coding style guidelines](https://meltpooldg.github.io/MeltPoolDG/howto.html).

## Bug reports
Reporting bugs is a valuable way to contribute to the further development of MeltPoolDG.
We keep track of all open issues with the library
[here](https://github.com/MeltPoolDG/MeltPoolDG-dev/issues). If you encounter a bug, please create an issue and,
whenever possible, include a minimal reproducible example that demonstrates the problem.
This greatly helps the developers investigate and resolve the issue efficiently.

## AI contributions

Only human beings can legally certify the <b>Developer Certificate of Origin</b>
listed above. Hence, at the present time, we are not accepting contributions
from autonomous AI agents. Patches may include source code created in part by an
AI tool as long as they satisfy the DCO and
1. The submitting author has personally read, reviewed, and understood the patch
   in its entirety.
2. The submitting author takes full responsibility for the contribution.

This is a rapidly evolving legal and technical field and these restrictions will
evolve over time. For more information, see [The Linux kernel's policy on AI
Coding Assistants](https://docs.kernel.org/next/process/coding-assistants.html)
